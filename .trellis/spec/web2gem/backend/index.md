# Backend Development Guidelines

> Cloudflare Worker TypeScript guidelines for the root `web2gem` package.

---

## Overview

The root package is a Cloudflare Worker that adapts Gemini Web behavior to OpenAI-compatible and Google-compatible HTTP APIs. Authored source lives under `src/`; `dist/worker.js` is generated by `pnpm build`.

---

## Pre-Development Checklist

- Read [Directory Structure](./directory-structure.md) before moving logic between modules.
- Read [Quality Guidelines](./quality-guidelines.md) before changing TypeScript types or replacing `any`.
- Read [Error Handling](./error-handling.md) before changing request parsing, SSE, or upstream error paths.
- Read [Logging Guidelines](./logging-guidelines.md) before adding logs.
- Read [Runtime Performance And Transport](./runtime-performance.md) before changing socket transport, config caching, bounded JSON reads, or stream delta coalescing.
- Read [Release Workflows](./release-workflows.md) before changing GitHub Actions release, Docker image, or release asset behavior.

---

## Validation Commands

Run these from `/workspace`:

```sh
pnpm check:static
pnpm typecheck
pnpm check:arch
pnpm unit
pnpm coverage:ci
pnpm smoke
```

`pnpm smoke` builds `dist/worker.js`, imports the bundle, checks public exports, and verifies the health route plus a DSML tool-call parse.

Run `pnpm check:worker-types` after Wrangler, binding, or Workers types changes;
run `pnpm check:bench` and `pnpm check:size` after runtime/build dependency or
hot-path changes.
